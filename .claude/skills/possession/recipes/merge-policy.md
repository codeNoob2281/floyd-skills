# 设定冲突处理策略

> 游戏角色可能存在不同版本设定差异，本策略规定如何记录和处理冲突。

## 常见冲突类型

### 1. 版本差异

同一角色在不同游戏版本中的设定变化：
- 初始版本 vs 后续版本
- 不同服务器版本（如国服/国际服）
- 改版/重做前后的设定

### 2. 来源差异

不同官方来源的设定表述：
- 游戏内文本 vs 官方Wiki
- 主线剧情 vs 活动剧情
- 游戏内 vs 衍生作品（漫画/动画）

### 3. 时间线差异

角色故事随时间发展的变化：
- 早期设定 vs 当前设定
- 不同章节的设定更新

## 处理原则

### 优先级排序

1. **最新官方设定优先**：以最新版本为准
2. **主线剧情优先**：主线>活动>衍生
3. **游戏内文本优先**：游戏内>外部资料

### 记录策略

**不强制统一**，而是记录差异：

```markdown
# 设定冲突：<角色名>

## 冲突1：[冲突主题]

### 版本/来源A
- [设定内容] （来源，时间）

### 版本/来源B
- [设定内容] （来源，时间）

### 处理建议
- 采用：[A/B/两者并存]
- 理由：...
```

## 冲突文件格式（写入 `conflicts.md`）

```markdown
# 设定冲突记录

## [冲突主题1]

### 差异描述
- 来源A说：...
- 来源B说：...

### 采用方案
- [选择] 及理由

---

## [冲突主题2]

### 差异描述
...

### 采用方案
...
```

## 何时写入 conflicts.md

- 同一设定有2个及以上不同表述
- 差异可能影响角色扮演的准确性
- 用户特别指出需要记录的冲突

## 何时不写入

- 明显的翻译/表述差异（语义相同）
- 非关键设定的细节差异
- 已有官方明确统一的冲突
