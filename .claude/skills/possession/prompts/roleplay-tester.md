# Prompt：扮演测试器

> 本模板引导AI自动测试角色扮演的一致性与准确性。
> 核心原则：**以角色设定为基准，检测 OOC（Out of Character）问题。**

## 输入

- **角色名**：`{name}`
- **已生成的维度文件**：
  - `profile.md`：角色档案
  - `personality.md`：性格与价值观
  - `interaction.md`：说话方式与台词
  - `memory.md`：背景故事
  - `relations.md`：人际关系

## 任务

扮演角色 `{name}`，完成 8 个测试场景的对话，并评估扮演一致性。

**核心约束**：
- 严格按照已提取的设定扮演角色
- 缺乏设定的内容，明确说明「这方面没有明确的设定」
- 不编造官方设定中不存在的信息

---

## 测试场景

### 场景 1：初次见面

**测试目标**：验证角色的自我介绍与社交风格

**测试输入**：
```
你好，请问你是谁？
```

**评估标准**：
- [ ] 是否使用了角色的自称（如「本水神」「大明星」）
- [ ] 语气是否符合角色的默认说话方式
- [ ] 是否有标志性表达或口头禅
- [ ] 是否符合角色的社交风格（主动/被动、开放/保留）

**评分标准**：
- 完全符合：1.0
- 基本符合：0.7
- 部分偏离：0.4
- 严重OOC：0.0

## 使用方式

### 在 Phase 5 之后执行

完成角色蒸馏后，自动运行扮演测试：

```
Phase 5：告知用户
    ↓
Phase 6：扮演测试（新增）
    ↓
输出测试报告
```

### 测试失败处理

- **通过率 < 60%**：提示用户「扮演测试未通过，建议补充设定材料」
- **关键 OOC 问题**：自动记录到 `issues.md`
- **轻微问题**：在测试报告中提供建议

---

## 注意事项

1. **测试输入应多样化**：根据角色特点定制，避免千篇一律
2. **评估应基于设定**：只检测是否符合已提取的设定，不主观判断
3. **区分「未知」与「OOC」**：
   - 角色说「这方面没有明确设定」—— 正确行为
   - 角色编造不存在的信息 —— OOC
4. **记录边界情况**：角色设定本身矛盾时，不应判定为 OOC
